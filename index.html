화성시 작은도서관 데이터 QA — 공공데이터 품질 검증
데이터 품질관리 허용값 기준 정의 NULL · 중복 검증 SQLite 적재 확인
목표

화성시 작은도서관 데이터를 대상으로 주소 기반 행정구역 기준을 정리하고, 정제된 데이터가 기준에 맞게 관리되는지 검증했습니다. 단순 분석보다 데이터의 누락·중복·허용값 오류를 확인하고 결과를 기록하는 QA 흐름에 초점을 두었습니다.

🔗 GitHub 보기
수행 내용
  • 전체 230건의 작은도서관 데이터를 대상으로 주소 기반 행정구역을 4개 허용값으로 표준화 매핑
  • region_gu 컬럼의 NULL 여부와 허용값 외 데이터 존재 여부를 pytest 기반 테스트 케이스로 검증
  • 의도적으로 잘못된 구 이름과 결측치를 주입하여 검증 로직이 실제 오류를 탐지하는지 확인
  • CSV 데이터를 SQLite에 적재한 뒤 SQL 조회로 행수와 정제 결과를 확인
  • 정제 전·후 데이터 상태와 검증 결과를 문서화하여 재확인 가능한 산출물로 정리
검증 결과
  • 전체 230건 기준 매핑 실패 0건 확인
  • 중복 데이터 0건 확인
  • region_gu NULL 0건 확인
  • 허용값 기준: 동탄구, 만세구, 효행구, 병점구
