프로젝트 개요

복약 관리, 감정 기록, AI 약물 상담을 하나의 흐름으로 연결한 AI 헬스케어 서비스입니다. 저는 이 프로젝트에서 감정 기록·일기·리포트 기능 구현과 사용자 입력 데이터가 날짜별로 저장·조회되는 흐름을 중심으로 기여했습니다.

담당 구현 범위
  • React/TypeScript 기반으로 날짜별 감정 기록 및 일기 작성 화면 구현
  • 선택 날짜, 작성 내용, 저장된 기록, 진료 배너 등 화면 상태 관리
  • 일기 저장·조회·수정·삭제 흐름을 백엔드 API와 연결
  • 저장된 감정 기록을 캘린더 및 일기 상세 화면과 연결
  • 일기 데이터를 기반으로 리포트 조회/생성 흐름 구현
  • 선택 날짜 기준 진료 배너 계산 로직 구현
  • 한국 시간 기준 날짜 처리로 today 판단 오류 개선

목표

수행 내용
  • 화성시 작은도서관 데이터 전체 230건을 대상으로 주소 기반 행정구역을 동탄구·만세구·효행구·병점구 4개 허용값으로 표준화 매핑
  • region_gu 허용값 검증 및 NULL 체크 — pytest 기반 테스트 케이스 자동화
  • 의도적으로 잘못된 구 이름·결측치를 주입한 뒤 검증 로직이 정확히 감지하는지 확인
  • SQLite 적재 후 조회 쿼리로 적재 결과와 정제 전·후 데이터 상태 비교 문서화
  • 전체 230건 기준 매핑 실패 0건, 중복 0건, region_gu NULL 0건 확인

Python pandas pytest SQLite CSV
